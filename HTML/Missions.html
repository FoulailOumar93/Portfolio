Mission GSB

Le projet Galaxy-Swiss Bourdin fournit aux visiteurs médicaux une interface pour saisir leurs frais et suivre leurs visites ; les responsables valident ensuite ces notes de frais. Stack : PHP / HTML5 / CSS / JavaScript / C# WinForms + base MySQL sur architecture MVC. Sécurité des données et UX ont été nos priorités.

Mission 1-B

Travaux solo : réalisation d’un formulaire qui affiche le cumul des frais d’un visiteur après sélection du mois et de l’année. Intégré à la page d’accueil, codé en MVC et stylisé en CSS, ce module m’a permis de prendre en main la stack GSB et de respecter la charte graphique.

Mission 2-B

Projet d’équipe (3 étudiants, 2ᵉ année) : application Web unifiée de gestion des frais pour plusieurs laboratoires. Mes rôles : conception + administration de la base de données et contribution à l’UI. Résultat : solution full-stack opérationnelle, facilitant le suivi des frais et l’export comptable.

Mission 3 – Projet Appli-CR

Objectif

Évolution du SI GSB : développement d’un module « Appli-CR » pour gérer médicaments, familles et dons. Travail mené en mode projet avec tests, déploiement et documentation complète.

Activités réalisées

  • A1.1.1 & A1.1.3 : analyse du cahier des charges et des exigences qualité
  • A1.3.1 & A1.3.4 : tests d’intégration / acceptation et déploiement
  • A4.1.6 → A4.1.10 : gestion env. dev/test, développement C#, tests unitaires, docs technique & utilisateur
  • A2.3.1 : qualification des anomalies et résolution

Cas d’utilisation principaux

Gérer les médicaments (secrétaire)
  1. Sélection d’une famille → affichage liste des médicaments
  2. Ajouter / modifier / supprimer un médicament (contrôles de cohérence)
  3. Créer une nouvelle famille si inexistante
Voir les médicaments offerts
  1. Sélection d’une famille → liste des dons triés par quantité
  2. Export possible au format XML

Ce module a nécessité la conception de pages sécurisées, la mise en place d’un tableau de bord, la génération de rapports XML et la rédaction de guides utilisateur/technique.
